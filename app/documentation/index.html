1. Preparing

Before you can run this code for both ios or android you will need to setup your computer to run react native

  1. You will need a MAC to compile for iphone(ios)
  2. I recommend MAC for android as well but you can still compile on windows for android
  3. You need an editor to edit the code files we recommend VSCode or Webstorm e.t.c
1. Installation

Do the following to complete the installation process

For MAC
  • Download Xcode latest version atleast 10.0 from here
  • Open a terminal install nodejs by typing brew install node (Make sure its node 10 or below )
  • ALso install watchman by typing in your terminal brew install watchman
  • Install react native with npm install -g react-native-cli
  • Install Yarn as well with brew install yarn
  • Create your first react native app by typing in terminal react-native init AwesomeProject
  • Now switch to the new app folder by typying cd AwesomeProject
  • Run your app with
    react-native run ios
  • If everything is success you should see your simulator with the react native default app welcome message else see the frequently ask questions before
  • But if you have problem, kindly read the full how to here at the official react native documentation under Building projects with Native Code
Run SoundKit App
  • Now download your SoundKit code from envato, extract to a folder on your computer.
  • In your terminal cd path/to/extracted/folder/app
  • Type in your terminal it can take up to 10minutes
    yarn install
  • Very Important for ios After this , kindly replace this file with the one included in your download file: node_modules/react-native/React/Base/RCTBridgeModule.h
  • Wait for it to complete after successful you run the app with either
    react-native run-ios
    or
    react-native run-android
  • Also if you notice slowness in the app on android you can always run the app in the release by using
    react-native run-android --variant=release
  • If the Ios does not show or close , you need to open the project in xcode and run from there , go to this folder ios/ and open SoundKit.xcodeproj

Configurations

As you run the app, you have use our demo api and website for rending the app, but you need to change it your website this section explain the configurations needed to achieve that

  • Make sure you are running the latest version 2.0.2
  • Go to your admin panel -> settings -> applications to set your own api key as shown below Make sure only alpha-numeric is used, no special character is allowed
  • Now open your SoundKit app code in an editor
  • Navigate to src/config.js to set your website url and other configurations
  • Replace 1 with your API Key you just set in your admin panel
  • Replace 2 with your site link for example https://mymusic.com/ make / end the website link
  • Go back and run your code from the terminal, the content of the app should now show your website contents

Facebook Setup

This is an advance section which require care to implement it , the project package name currently is com.krea8iv.SoundKit and that is what the demo is using, which you need to change to yours something like com.domain.appName

You can rename your project with the following instructions

For Android

  • Open your terminal and locate your SoundKit code folder and run yarn global add react-native-rename
  • Run this to rename to your package react-native-rename "App Name" -b com.domain.appName
  • Make sure to change App Name to your app name
  • Also change the domain to your domain and appName to your app name without space

You can do this in the Xcode see below
